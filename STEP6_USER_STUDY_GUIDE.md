## Conducting the Study

### Step 1: Navigate to User Study Page

1. Open the app (it opens automatically in browser)
2. Click **"💬 User Study"** in the left sidebar

### Step 2: Load Configuration

The configuration section is expanded by default on first visit.

**Default paths (should work without changes):**

```
Selected Queries CSV: outputs/user_study/workflow_demo/selected_queries_for_user_study.csv
Trained Model: experiments/banking77/banking77_logistic_model.pkl
Hierarchy JSON: config/hierarchies/banking77_hierarchy.json
Intents JSON: config/hierarchies/banking77_intents.json
Optimal Thresholds: results/banking77/workflow_demo/banking77_optimal_thresholds.json
Output Directory: outputs/user_study/human_study
```

Click **"📥 Load Configuration"** button.

✅ You should see success messages for each loaded component.

### Step 3: Process Queries

**For each query:**

1. **Read the query** displayed in the blue info box
2. **Check belief progression** - Top 5 intent candidates with confidence scores
3. **Optional: View ground truth** - Expand "👁️ Ground Truth" to see true intent
4. **Handle clarification** (if system asks):
   - Read the chatbot's question
   - Type your natural language response
   - Click **"✅ Submit Response"**
   - System updates beliefs and may ask follow-up questions
5. **Validate prediction** (when system is confident):
   - Review predicted intent and confidence
   - Click **"✅ Correct Prediction"** or **"❌ Incorrect Prediction"**

**Progress automatically advances** to the next query after validation.

### Step 4: Session Management

**During the study:**

- **💾 Save Progress**: Creates checkpoint with current results
- **🔄 Reset Session**: Restart from query 1
- **Session Controls** are in the expandable section at the bottom

**After completing all queries:**

- Session summary table displays all results
- Click **"📥 Download Session Data"** to save CSV
- Click **"🔄 Start New Session"** to process queries again (e.g., with different user)

## Output Files

Session results are saved to:

```

```

### CSV Columns

| Column | Example | Description |
|--------|---------|-------------|
| `query_idx` | 0 | Query position in study (0-based) |
| `query` | "I want to activate my card" | Original user query |
| `true_intent` | `activate_my_card` | Ground truth label |
| `predicted_intent` | `activate_my_card` | DS agent's prediction |
| `confidence` | 0.7530 | Final confidence score |
| `is_correct` | True | Whether prediction matches truth |
| `num_clarifications` | 2 | Number of dialogue turns |
| `conversation` | "Chatbot: ...\nUser: ..." | Full dialogue history |
| `timestamp` | "2026-02-05T14:30:15.123456" | When validated |

## Troubleshooting

### "❌ Queries file not found"

**Cause:** STEP 5 hasn't been run yet.

**Solution:**
1. Open `notebooks/system_workflow_demo.ipynb`
2. Run cell 14 (STEP 5: Select User Study Queries)
3. Verify file exists: `outputs/user_study/workflow_demo/selected_queries_for_user_study.csv`

### "❌ Error loading configuration: FileNotFoundError"

**Cause:** Missing trained model or config files.

**Solution:**
1. Run STEP 0 (cell 5) to train the model
2. Run STEP 2 (cell 9) to compute thresholds
3. Verify files exist:
   - `experiments/banking77/banking77_logistic_model.pkl`
   - `results/banking77/workflow_demo/banking77_optimal_thresholds.json`

### "❌ Error loading configuration: KeyError 'query'"

**Cause:** Selected queries CSV has wrong format.

**Solution:**
1. Re-run STEP 5 (cell 14)
2. Check CSV has columns: `query`, `true_intent`, `num_interactions`, `confidence`

### Interface not responding

**Solutions:**
1. Refresh browser page
2. Check terminal for errors
3. Restart Streamlit: `Ctrl+C` then re-run `streamlit run app_main.py`

### Belief progression shows zeros

**Cause:** DSMassFunction not computing beliefs correctly.

**Solution:**
1. Check that `enable_belief_tracking=True` (automatically set by app)
2. Verify intent embeddings loaded correctly (check success messages)

## Tips for Running User Studies

### For Researchers

- **Pilot test**: Process 5-10 queries yourself before recruiting participants
- **Instructions**: Show participants this guide before starting
- **Session length**: 100 queries ≈ 30-45 minutes (depending on clarification rate)
- **Breaks**: Use "💾 Save Progress" to allow participants to take breaks

### For Participants

- **Be natural**: Respond to clarifications as you would in real conversation
- **Don't overthink**: Your first instinct is usually correct
- **Ask for help**: If unclear what the system wants, type a question
- **Save often**: Use "💾 Save Progress" every 20-30 queries

### Multiple Sessions

To process the same queries with different users:

1. Complete first user's session → Download data
2. Click "🔄 Start New Session"
3. Process queries again (query order remains the same)
4. Each session generates a new timestamped CSV file
